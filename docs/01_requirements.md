## 1. プロジェクト概要

### 1.1 目的

Vision AI Scanner は、カメラ映像からリアルタイムでテキスト抽出（OCR）および物体検出を行う Web アプリケーションである。Google Cloud Vision API をバックエンドに使用し、ブラウザ上でカメラを起動して映像を安定検知した後に自動撮影・解析を行う。

### 1.2 対象ユーザー

- カメラ映像から文字を読み取りたいユーザー（書類、看板、名刺など）
- 映像内の物体を識別・分類したいユーザー（産業検査、学習用途など）

### 1.3 解決する課題

| 課題 | 解決方法 |
|------|----------|
| 手動での画像撮影・アップロードの手間 | カメラの静止検知による自動撮影 |
| OCR結果の精度不足 | 画像前処理（コントラスト・シャープネス強調）による精度向上 |
| API利用コストの無制御な増大 | クライアント側・サーバー側の二重レート制限 |
| 英語ラベルの理解しづらさ | 物体検出ラベルの英日翻訳辞書による日本語表示 |

## 2. 機能要件

### 2.1 テキスト抽出（OCR）

| 項目 | 仕様 |
|------|------|
| 機能概要 | カメラに映した文字をリアルタイムで読み取る |
| API機能 | Google Cloud Vision API `DOCUMENT_TEXT_DETECTION` |
| 言語ヒント | 英語（`en`）優先、日本語（`ja`）補助 |
| 前処理 | Pillow によるコントラスト強調（1.5倍）+ シャープネス強調（1.5倍） |
| 出力形式 | 検出テキスト一覧（ラベル + バウンディングボックス座標） |
| 画面表示 | 映像上に緑色のバウンディングボックスを描画、結果パネルにタイムスタンプ付きテキスト表示 |

### 2.2 物体検出

| 項目 | 仕様 |
|------|------|
| 機能概要 | 映像内の物体を識別し日本語ラベル付きで表示する |
| API機能 | Google Cloud Vision API `OBJECT_LOCALIZATION` |
| 最大検出数 | 10件 |
| 翻訳 | `translations.py` による英日翻訳辞書（約120語） |
| 出力形式 | ラベル（英名+日本語名）+ 信頼度スコア + バウンディングボックス（正規化座標 0〜1） |
| 画面表示 | 映像上に赤色のバウンディングボックス + ラベル表示、結果パネルにタイムスタンプ付き表示 |

### 2.3 静止検知自動撮影

| 項目 | 仕様 |
|------|------|
| 機能概要 | カメラが安定した（動きが止まった）ことを検知して自動撮影する |
| 検出方式 | フレーム間差分（`requestAnimationFrame` ループ） |
| 差分計算解像度 | 64x48ピクセル（低解像度で高速計算） |
| モーション閾値 | ピクセルあたり平均差分 30 未満で安定と判定 |
| 安定判定フレーム数 | 30フレーム連続安定（約1秒 @30fps） |
| キャプチャ領域 | ターゲットボックス内（映像の60%領域）のみクロップして送信 |
| キャプチャ品質 | JPEG 95%品質 |
| 進捗表示 | 安定化プログレスバーで視覚的にフィードバック |

### 2.4 API使用量管理

| 項目 | 仕様 |
|------|------|
| 機能概要 | API呼び出し回数をクライアント側・サーバー側の二重構造で管理する |
| **サーバー側制限** | IP単位のレート制限（分間上限 + 日次上限） |
| デフォルト上限 | 20回/分、1000回/日（環境変数で変更可能） |
| バックエンド | Redis（原子操作/Luaスクリプト）またはインメモリ（TTLCache）の自動選択 |
| 制限時レスポンス | HTTP 429 + エラーメッセージ |
| 失敗時の回収 | API呼び出しが失敗した場合、予約済みカウントを取り消す（`release_request`） |
| **クライアント側管理** | `localStorage` で日次カウントを記録・表示 |
| 日次上限 | サーバーの `/api/config/limits` から動的取得 |
| 警告表示 | 使用率80%で黄色、100%で赤色表示 |
| カウント加算条件 | API呼び出し成功時のみカウント（失敗時は加算しない） |

### 2.5 レスポンシブUI

| 項目 | 仕様 |
|------|------|
| 機能概要 | PC・モバイル両対応のWebインターフェース |
| デザイン | Material Design 3 ダークテーマ、Glassmorphism風パネル |
| レイアウト | 左パネル（映像）+ 右パネル（結果）のグリッドレイアウト |
| フォント | Noto Sans JP（日本語）+ Inter（英数字） |
| モード切替 | テキスト / 物体検出をボタンで即座に切り替え可能 |
| カメラ切替 | 複数カメラデバイスがある場合は切替ボタンを表示 |
| ミラー（左右反転） | インカメラ使用時に映像を左右反転可能 |
| ファイル入力 | 動画ファイルをアップロードして解析可能 |

### 2.6 管理API

| 項目 | 仕様 |
|------|------|
| 機能概要 | プロキシ設定の確認・変更を行う管理用API |
| 認証方式 | `X-Admin-Secret` ヘッダーによるシークレット認証 |
| エンドポイント | `GET /api/config/proxy`（設定確認）、`POST /api/config/proxy`（設定変更） |
| 未認証時 | `GET` はON/OFFのみ返却、`POST` は常に403 |

### 2.7 ヘルスチェック

| エンドポイント | 用途 | 正常時 | 異常時 |
|---------------|------|--------|--------|
| `GET /healthz` | Liveness（プロセス生存確認） | `200 {"status": "ok"}` | 応答なし |
| `GET /readyz` | Readiness（処理可能確認） | `200 {"status": "ok"}` | `503 {"status": "not_ready"}` |

## 3. 非機能要件

### 3.1 性能

| 項目 | 要件 |
|------|------|
| API応答時間 | 15秒以内（タイムアウト設定） |
| フロントエンド応答 | `requestAnimationFrame` ベースで60fpsを目標 |
| フレーム差分計算 | 64x48ピクセルの低解像度キャンバスで高速処理 |
| HTTP接続管理 | `requests.Session` による接続プール再利用 |
| 画像サイズ上限 | 5MB（Base64デコード後） |
| リクエストボディ上限 | 10MB（Base64 + JSONオーバーヘッド） |
| 画像ピクセル上限 | 2000万ピクセル（展開爆弾対策） |
| 静的ファイルキャッシュバスティング | MD5ハッシュによるバージョン付与（`?v=xxxxxxxx`） |

### 3.2 セキュリティ

| 項目 | 要件 |
|------|------|
| **Content Security Policy (CSP)** | `script-src 'self'`、`style-src 'self' 'nonce-...'`（unsafe-inline完全排除） |
| **CSPノンス** | リクエストごとに `secrets.token_urlsafe(16)` で生成 |
| **CORS** | デフォルト同一オリジンのみ。`ALLOWED_ORIGINS` 環境変数で明示的に許可 |
| **相関ID** | 全リクエストに `X-Request-Id` ヘッダー付与（ログ追跡用） |
| **MIME検証** | JPEG/PNGのmagic byte検証（Base64デコード後に実バイナリを検査） |
| **Base64検証** | `base64.b64decode(validate=True)` による厳密デコード検証 |
| **XSS対策** | フロントエンドで `innerHTML` 禁止、`textContent`/`createTextNode` による安全なDOM操作 |
| **イベントリスナー** | HTML inline属性（onclick, style）禁止 → JSの `addEventListener` で登録 |
| **X-Content-Type-Options** | `nosniff`（MIMEスニッフィング防止） |
| **X-Frame-Options** | `DENY`（クリックジャッキング防止） |
| **Referrer-Policy** | `strict-origin-when-cross-origin` |
| **Permissions-Policy** | `camera=(self), microphone=(self)`（カメラ・マイクの同一オリジン限定） |
| **Cache-Control** | `no-store, no-cache, must-revalidate, max-age=0`（キャッシュ無効化） |
| **APIキー送信方式** | ヘッダー（`x-goog-api-key`）で送信（URLパラメータに含めない） |
| **プロキシURL認証情報** | ログ出力時にマスク処理（`user:pass` → `***:***`） |
| **ADMIN_SECRET** | 起動時にエントロピーチェック（文字種3種以上、16文字以上を推奨） |
| **管理API認証** | `X-Admin-Secret` ヘッダー照合、未設定時は常に403 |

### 3.3 可用性

| 項目 | 要件 |
|------|------|
| レート制限バックエンド | Redis（マルチプロセス安全）→ インメモリ（フォールバック）の自動切替 |
| Redis障害時 | インメモリに自動フォールバック（`/readyz` で警告を返却） |
| APIリトライ | 3回、指数バックオフ（0.5秒基準）、429/500/502/503で再試行 |
| フロントエンドエラー回復 | エラー後5秒待機して自動再試行（`scheduleRetry`） |
| カメラフォールバック | 指定デバイスでの取得失敗時、deviceIdなしでリトライ |
| 画像前処理フォールバック | 前処理失敗時はスキップして元画像のまま送信（安全チェック違反を除く） |
| ヘルスチェック | `/healthz`（Liveness）+ `/readyz`（Readiness）の2階層 |
| 画面離脱時 | `visibilitychange` イベントでカメラストリームを自動停止 |

### 3.4 保守性

| 項目 | 要件 |
|------|------|
| 構造化ログ | `event=xxx request_id=xxx` 形式のログ出力 |
| ログレベル | `INFO`（デフォルト）、`FLASK_DEBUG=true` で `DEBUG` |
| APIレスポンスログ | エラー時500文字に制限 |
| コードスタイル | ruff によるリンティング |
| セキュリティスキャン | bandit による静的解析 |
| 依存関係監査 | pip-audit による脆弱性チェック |
| CI/CD | GitHub Actions（pytest + ruff + bandit + pip-audit + detect-secrets） |
| 依存管理 | Dependabot（pip依存 + GitHub Actions 週次チェック） |
| テスト | pytest（単体テスト + APIテスト）+ Playwright（E2E） |
| 環境変数管理 | `.env` ファイル（`.gitignore` 対象）、`.env.example` はコミット |

## 4. 制約事項

### 4.1 対応ブラウザ

| ブラウザ | 対応状況 | 備考 |
|---------|---------|------|
| Google Chrome | 対応 | 推奨ブラウザ |
| Microsoft Edge | 対応 | Chromiumベース |
| Mozilla Firefox | 対応 | — |
| Safari | 未検証 | カメラAPI互換性に注意 |

### 4.2 HTTPS要件

- カメラアクセス（`navigator.mediaDevices.getUserMedia`）には HTTPS または `localhost` が必須
- 開発環境では `http://localhost:5000` で動作可能
- 本番環境ではHTTPSを必須とする

### 4.3 API上限

| 制限 | デフォルト値 | 変更方法 |
|------|------------|----------|
| 分間リクエスト上限 | 20回/分（IP単位） | 環境変数 `RATE_LIMIT_PER_MINUTE` |
| 日次リクエスト上限 | 1000回/日（IP単位） | 環境変数 `RATE_LIMIT_DAILY` |
| 画像サイズ上限 | 5MB（Base64デコード後） | ソースコード内定数 `MAX_IMAGE_SIZE` |
| リクエストボディ上限 | 10MB | ソースコード内定数 `MAX_REQUEST_BODY` |
| Vision API最大結果数 | 10件 | ソースコード内定数 `MAX_RESULTS` |
| APIタイムアウト | 15秒 | ソースコード内定数 `API_TIMEOUT_SECONDS` |

### 4.4 言語制約

- OCR言語ヒントは英語（`en`）に最適化されており、日本語テキストの認識精度は限定的
- 物体検出ラベルの翻訳辞書は約120語のみ対応。未登録の英語ラベルはそのまま表示

### 4.5 インフラ制約

- Redis未接続時はインメモリフォールバック（プロセス再起動でカウントリセット）
- マルチプロセス運用（gunicorn等）では Redis が必須
- APIキーは起動時に1回のみ読み込み（動的再読込は非対応）

## 5. ユースケース一覧

### UC-01: テキスト抽出スキャン

| 項目 | 内容 |
|------|------|
| アクター | ユーザー |
| 前提条件 | カメラアクセスが許可されている、APIキーが設定されている |
| 基本フロー | 1. アプリを開く → 2. モード「テキスト」を選択 → 3.「スタート」ボタンを押す → 4. カメラを文字に向けて約1秒静止 → 5. 安定化バーが100%に達する → 6. ターゲットボックス内の映像が自動キャプチャされる → 7. APIで解析 → 8. 検出テキストが緑色バウンディングボックスと共に表示される → 9. 結果パネルにタイムスタンプ付きで追加される |
| 代替フロー | カメラが動いている場合は安定化バーがリセットされ、静止するまで撮影されない |
| 例外フロー | API呼び出し失敗時は5秒待機後に自動再試行。レート制限到達時はエラーメッセージ表示 |

### UC-02: 物体検出スキャン

| 項目 | 内容 |
|------|------|
| アクター | ユーザー |
| 前提条件 | カメラアクセスが許可されている、APIキーが設定されている |
| 基本フロー | 1. モード「物体」に切り替え → 2.「スタート」ボタンを押す → 3. カメラを物体に向けて約1秒静止 → 4. 自動キャプチャ → 5. 検出された物体が赤色バウンディングボックス + 日本語ラベル + 信頼度スコアで表示される |
| 特記事項 | 翻訳辞書にない英語ラベルはそのまま表示される |

### UC-08: システムヘルスチェック（運用者）

| 項目 | 内容 |
|------|------|
| アクター | 運用者 / ロードバランサー |
